Understanding Scopes in JavaScript

Scope is a concept that can be confusing and frustrating when learning for the first time but is enormously useful once mastered. It allows for flexibility by isolating variables based on their lexical environment. For example, writing a program in which all variables are located in the same scope would require that each variable be unique to avoid overwriting and redefining variables. Thus, by breaking up code into smaller pieces lexical scoping allows code to be read section by section, and not have to worry about all the code at once. Scope only arises with the use of functions, and in its most simple form can be thought of as the visibility of variables.
Variables: Local Vs. Global
A variable declared outside of any function is called a global variable and all other functions have access to this global variable. A variable that is declared inside of a function using the “var” key word is only defined within that function and called a local variable. Local variables are undefined outside of their scope, while global variables are defined everywhere.

Closures
When you access a variable outside of its scope it is called a closure. Stated differently, a closure is a function that is out of scope but still has access to the outer scopes’ variables.
